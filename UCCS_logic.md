# UCCS 插件邏輯完整說明

> Unified Chinese-characters Searcher (UCCS) v2.0.0

---

## 🎯 核心功能

在 Obsidian 全局搜尋（`Cmd+Shift+F`）中，自動將輸入的中文字元擴展為多種地區寫法，確保輸入任何一個中文字都能找到所有對應中文的結果。

---

## 🧩 名詞解釋

| 名詞 | 說明 | 範例 |
|------|------|------|
| **原文** | 用戶在搜索欄輸入的文字 | `老板` |
| **字元轉換** | 逐字查映射表，找出該字的繁/簡變體 | `丝` → `絲` |
| **短語補充** | 整詞查短語表，補上字符轉換無法處理的結果 | `老板` → `老闆` |
| **格子** | 展開結果中的每個 `(文字)` 區塊 | `(老板) OR (老闆)` 有 2 格 |
| **防抖** | 等待用戶停頓後才展開，避免打字中途誤展開 | 預設 800ms |
| **隱式模式** | 展開後立即恢復原文，用戶看不到展開內容 | 僅後端搜尋 |

├──────────────────────────────────────────────────┤
│                                                   │

│                                                   │

│                                                   │

│                                                   │

## 📦 _expandQuery 展開邏輯（核心）

```
輸入: 原文（如「老板」）
    │
    ▼
┌──────────────────────────────────────────┐
│           tokenize 分詞                   │
│                                          │
│  將輸入拆成 token 陣列                     │
│  支援: plain / operator / open / close   │
│  / or / quoted / whitespace              │
│                                          │
│  例如: "tag:动漫"                          │
│  → [{type:'operator', raw:'tag:动漫'}]   │
│                                          │
│  例如: "剑法"                              │
│  → [{type:'plain', raw:'剑法',           │
│       value:'剑法'}]                      │
│                                          │
└──────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────┐
│         逐個 token 處理                   │
├──────────────────────────────────────────┤
│                                          │
│  ┌─────────────────────────────┐         │
│  │  type = 'plain' 且有中文？   │         │
│  │  是 → 執行字元轉換 + 短語    │         │
│  │  否 → 保留 raw，跳過         │         │
│  └─────────────────────────────┘         │
│                                          │
│  ┌─────────────────────────────┐         │
│  │  type = 'operator' 且        │         │
│  │  keepOperators = true？      │         │
│  │  是 → 轉換運算符後的值        │         │
│  │  否 → 保留 raw，跳過         │         │
│  └─────────────────────────────┘         │
│                                          │
│  其他 type → 保留 raw                    │
│                                          │
└──────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────┐
│         組裝展開結果                      │
│                                          │
│  所有 token 處理完後，用空格 join          │
│                                          │
│  例如:                                    │
│  plain "剑法" → "(剑法) OR (劍法)"       │
│  空格 " "                                │
│  plain "入门" → "(入门) OR (入門)"       │
│                                          │
│  結果:                                    │
│  "(剑法) OR (劍法) (入门) OR (入門)"     │
│                                          │
└──────────────────────────────────────────┘
```

## 🔤 字元轉換（詳解）

### 單字元

```
輸入: 「里」
    │
    ▼
getVariants("里")
    │
    ├── HK 模式: s2hk["里"] = "裏" → ["裏"]
    │   結果: (里) OR (裏)
    │
    ├── TW 模式: s2tw["里"] = "裡" → ["裡"]
    │   結果: (里) OR (裡)
    │
    ├── ALL 模式: s2all["里"] = ["裏", "裡"] → ["裏", "裡"]
    │   結果: (里) OR (裏) OR (裡)
    │         └簡─┘  └HK─┘  └TW─┘
    │
    └── TW-HK 模式: hk_tw["里"] = undefined → []
        結果: (里) ← 無變體，不展開
```

**實例：**

| 輸入 | HK | TW | ALL | TW-HK |
|------|----|----|-----|-------|
| `龙` | `(龙) OR (龍)` | `(龙) OR (龍)` | `(龙) OR (龍)` | 不展開 |
| `烟` | `(烟) OR (菸)` | `(烟) OR (煙)` | `(烟) OR (煙)` | 不展開 |
| `启` | `(启) OR (啓)` | `(启) OR (啟)` | `(启) OR (啓) OR (啟)` | 不展開 |
| `裏` | 不展開（已是繁體） | 不展開（TW 用裡） | 不展開 | 不展開（兩地相同） |

**繁體輸入：**

| 輸入 | HK | TW | ALL | TW-HK |
|------|----|----|-----|-------|
| `龍` | `(龍) OR (龙)` | `(龍) OR (龙)` | 不展開（無 t2s） | 不展開 |
| `過` | `(過) OR (过)` | `(過) OR (过)` | 不展開 | 不展開 |
| `門` | `(門) OR (门)` | `(門) OR (门)` | 不展開 | 不展開 |
| `啓` | 不展開（已是HK） | `(啓) OR (啟)` | 不展開 | `(啓) OR (啟)` |
| `啟` | `(啟) OR (啓)` | 不展開（已是TW） | 不展開 | `(啟) OR (啓)` |
| `裏面` | `(裏面) OR (里面)` | 不展開（TW用裡） | 不展開 | 不展開 |

### 多字元 — 非 ALL 模式（HK / TW）

```
輸入: 「剑法」
    │
    ▼
字元轉換（逐字取 v[0]）：
  剑 → s2hk["剑"] = "劍" → v[0] = "劍"
  法 → s2hk["法"] = undefined → 保留 "法"
    │
    ▼
轉換結果: "劍法"
    │
    ▼
"劍法" !== "剑法" → 加到 allAlts
    │
    ▼
terms = ["剑法", "劍法"]
    │
    ▼
短語（如開啟）: stPhrases["剑法"]? → 無
    │
    ▼
結果: (剑法) OR (劍法)
```

**實例：**

| 輸入 | HK | TW |
|------|----|----|
| `剑法` | `(剑法) OR (劍法)` | `(剑法) OR (劍法)` |
| `龙腾` | `(龙腾) OR (龍騰)` | `(龙腾) OR (龍騰)` |
| `电脑` | `(电脑) OR (電腦)` | `(电脑) OR (電腦)` |
| `老板` | 不展開（無變體） | 不展開（無變體） |

### 多字元 — ALL 模式

```
輸入: 「里里外外」
    │
    ▼
字元轉換（HK 版，取 v[0]）：
  里 → s2all["里"] = ["裏", "裡"] → v[0] = "裏"
  里 → v[0] = "裏"
  外 → 無變體 → "外"
  外 → 無變體 → "外"
    │
    ▼
HK 版: "裏裏外外" ≠ "里里外外" → 加到 allAlts
    │
    ▼
字元轉換（TW 版，取 v[1]，因 region === 'all'）：
  里 → v[1] = "裡"
  里 → v[1] = "裡"
  外 → "外"
  外 → "外"
    │
    ▼
TW 版: "裡裡外外"
"裡裡外外" !== "里里外外" 且不在 allAlts → 加到 allAlts
    │
    ▼
terms = ["里里外外", "裏裏外外", "裡裡外外"]
    │    └簡體─┘  └繁HK──┘  └繁TW──┘
    ▼
短語（如開啟）: stPhrases["里里外外"] = "裏裏外外"
"裏裏外外" 已在 terms 中 → 跳過（不重複）
    │
    ▼
結果: (里里外外) OR (裏裏外外) OR (裡裡外外)
```

**實例：**

| 輸入 | ALL 結果 | 格子說明 |
|------|---------|---------|
| `里里外外` | `(里里外外) OR (裏裏外外) OR (裡裡外外)` | 簡 + 繁HK + 繁TW |
| `剑法` | `(剑法) OR (劍法)` | 簡 + 繁（HK 和 TW 同字） |
| `老板` | 不展開 | 老和板均無變體 |

### 多字元 — TW-HK 模式

```
輸入: 「啓」
    │
    ▼
hk_tw["啓"] = "啟" → ["啟"]
    │
    ▼
terms = ["啓", "啟"]
    │
    ▼
結果: (啓) OR (啟)
```

**實例：**

| 輸入 | TW-HK 結果 | 說明 |
|------|-----------|------|
| `啓` | `(啓) OR (啟)` | HK寫法 vs TW寫法 |
| `兑` | `(兑) OR (兌)` | 80 個差異字之一 |
| `裏` | 不展開 | 兩地寫法相同 |
| `龙` | 不展開 | 簡體字，TW-HK 不處理 |

## 📋 短語補充模組

### 流程

```
字元轉換完成後
    │
    ▼
短語轉換開啟？──┴──→ 跳過
    │
    ▼
查 stPhrases[文字]（簡→繁短語表）
    │
    ├── 找到 → variant = 短語結果
    └── 沒找到 → 查 tsPhrases[文字]（繁→簡短語表）
         │
         ├── 找到 → variant = 短語結果
         └── 沒找到 → 跳過
              │
              ▼
variant 已在 terms 中？
    │
    ├── 是 → 跳過（不重複）
    └── 否 → 追加到 terms
```

### 三種情況

| 情況 | 字元轉換 | 短語 | 結果 | 範例 |
|------|---------|------|------|------|
| **① 短語補上** | 無結果 | 有結果 | 追加 | `老板` → `+ (老闆)` |
| **② 短語跳過** | 有結果 | 相同 | 不追加 | `一丝不挂` → 兩者都是 `一絲不掛` |
| **③ 短語追加** | 有結果 | 不同 | 追加 | TW `里里外外` → 字元`裡裡外外` + 短語`裏裏外外` |

### 實例對照

| 輸入 | 模式 | 短語關 | 短語開 | 差別 |
|------|------|--------|--------|------|
| `老板` | HK | 不展開 | `+ (老闆)` | ⭐ 短語救了它 |
| `老板` | TW | 不展開 | `+ (老闆)` | ⭐ 同上 |
| `老板` | ALL | 不展開 | `+ (老闆)` | ⭐ 同上 |
| `一丝不挂` | HK | `(一丝不挂) OR (一絲不掛)` | 同左 | 短語結果相同，跳過 |
| `一丝不挂` | ALL | `(一丝不挂) OR (一絲不掛)` | 同左 | 短語結果相同，跳過 |
| `里里外外` | TW | `(里里外外) OR (裡裡外外)` | `+ (裏裏外外)` | 短語不同，追加 |
| `叶公好龙` | HK | `(叶公好龙) OR (葉公好龍)` | 同左 | 短語相同，跳過 |
| `自行车` | HK | `(自行车) OR (自行車)` | 同左 | 短語相同，跳過 |
| `乱七八糟` | HK | `(乱七八糟) OR (亂七八糟)` | 同左 | 短語相同，跳過 |

## 🔧 設定項一覽

### 基本設置

| 設定 | 說明 | 預設值 | 可選值 |
|------|------|--------|--------|
| 啟用插件 | 一鍵開關 | 開啟 | 開/關 |
| 映射地區 | 簡繁轉換的地區變體 | 香港（HK） | HK/TW/ALL/TW-HK |

### 高級功能

| 設定 | 說明 | 預設值 |
|------|------|--------|
| 轉換運算符值 | 轉換 path:、tag: 等搜尋運算符後的值 | 開啟 |
| 展開延遲 | 打字停頓後多少 ms 展開（200~2000ms） | 800ms |
| 簡化搜索欄模式 | 搜尋欄只顯示原文，不顯示展開結果 | 關閉 |
| 短語/成語補充轉換 | 字符轉換無法處理時補充短語級轉換 | 關閉 |

### 4 種地區模式說明

| 模式 | 匹配方式 | 範例 |
|------|---------|------|
| 簡體↔繁體HK | `烟 / 菸` → `(烟) OR (菸)` |
| 簡體↔繁體TW | `启 / 啟` → `(启) OR (啟)` |
| 全部地區 | `里 / 裏 / 裡` → `(里) OR (裏) OR (裡)` |
| *繁體HK↔繁體TW | `裏 / 裡` → `(裏) OR (裡)` |

## 🧪 邊界案例

### 輸入法處理

```
用戶用拼音輸入「劍」：
  1. 用戶按 j → compositionstart 觸發
  2. 用戶按 i → 組合中，所有 input 跳過
  3. 用戶按 a → 組合中
  4. 用戶按 n → 組合中
  5. 用戶選字 → compositionend 觸發
  6. 搜索欄顯示「劍」
  7. _processInput 正常執行
```

### 展開後追加文字

```
用戶先輸入「杖与」→ 展開為 (杖与) OR (杖與)
用戶繼續輸入「剑」→ 搜索欄變為 (杖与) OR (杖與)剑
    │
    ▼
_isExpansionWithExtra 檢測到：
  ┌─────────────────────────────────┐
  │ 正則: /(\([^)]+\)\s+OR\s+\([^)]+\))/  │
  │ 匹配到: (杖与) OR (杖與)                  │
  │ before = "" (展開前無文字)               │
  │ after = "剑" (展開後追加的文字)           │
  │ !!("" || "剑") = true → 處理            │
  └─────────────────────────────────┘
    │
    ▼
_stripExpansion → 剝離展開部分
  original = "杖与"（從展開字串還原）
  after = "剑"
  結果: "杖与剑"
    │
    ▼
觸發 input 事件 → 重新處理「杖与剑」
    │
    ▼
800ms 後 → 展開為 (杖与剑) OR (杖與劍)
```
